Inicio

Apache Tomcat, también conocido simplemente como Tomcat, es un servidor de aplicaciones web de código abierto desarrollado por la Apache Software Foundation. Su función principal es implementar las tecnologías Java Servlet, JavaServer Pages (JSP) y Java Expression Language (EL), proporcionando un entorno para ejecutar aplicaciones web basadas en Java.

Características

  1. Servidor Web y Contenedor de Servlets: Tomcat funciona tanto como un servidor web como un contenedor de servlets, permitiendo la ejecución de aplicaciones web Java. Los servlets son programas Java que extienden la capacidad de los servidores web para manejar solicitudes y respuestas, facilitando el desarrollo de aplicaciones dinámicas.

  2. Soporte para JSP: Tomcat también es compatible con JavaServer Pages (JSP), una tecnología que simplifica la creación de páginas web dinámicas mediante la incorporación de fragmentos de código Java directamente en las páginas HTML.

  3. Ligero y Eficiente: Tomcat se destaca por su ligereza y eficiencia, lo que lo convierte en una opción popular para implementaciones que requieren un servidor web ágil y fácil de configurar.

  4. Código Abierto: Como proyecto de código abierto bajo la Apache License, Tomcat es accesible y modificable por la comunidad, fomentando la colaboración y el desarrollo continuo.

Instalación

  1. En este manual vamos a ver cómo podemos instalar Apache Tomcat para nuestro sistema Linux y sus configuraciones básicas para poder manejarlo con mayor comodidad. En este caso estaremos usando Ubuntu.

  2. Lo primero que haremos será hacer una actualización del sistema por completo, para ello usaremos el siguiente comando:

    $ sudo apt update

  3. A continuación, agregaremos un nuevo repositorio de software llamado "linuxuprising/java" al sistema. Este repositorio específico generalmente se utiliza para proporcionar versiones más recientes del entorno de ejecución de Java (JRE) y del Kit de Desarrollo de Java (JDK) en comparación con las versiones que se incluyen en los repositorios oficiales de Ubuntu. Para ello usaremos el siguiente comando:

    $ sudo add-apt-repository ppa:linuxuprising/java

  4. Una vez añadido el repositorio lo siguiente que debemos hacer es actualizar el sistema de nuevo, para ello repetiremos el primer paso:

    $ sudo apt update

  5. Ahora vamos a proceder a instalar la versión de Java que deseemos, en este caso vamos a instalar la versión 17 (JDK 17).

    $ sudo apt install oracle-java17-installer

    Nos deberá salir la siguiente ventana donde le daremos a aceptar:

    Y en esta le daremos a si:

  6. Una vez hayamos terminado el paso anterior, vamos a actualizar de nuevo el sistema con el siguiente comando:

    $ sudo apt update

  7. Ahora debemos comprobar que se haya instalado correctamente. Para ello usaremos:

    $ java -version

    Como podemos observar, tenemos la versión 17 de Java (JDK 17)

  8. Para poner esta versión de Java como versión predeterminada usaremos el siguiente comando:

    $ sudo apt install oracle-java17-set-default

  9. Una vez hemos instalado Java procederemos a instalar Apache Tomcat cuyo servidor será el que interprete el código Java. Para ello usaremos:

    $ sudo apt install tomcat9 tomcat9-admin

  10. Cuando termine la descarga deberemos comprobar en que estado se encuentra nuestro Apache Tomcat, para ello usaremos:

    $ sudo systemctl status tomcat9

    Si nos aparece que no está activo, es porque no tenemos el JDK instalado, ya que nosotros hemos instalado solamente Java. Para poder solucionar este error, solamente debemos instalar el JDK a través de:

    $ sudo apt install default-jdk

    Acto seguido reiniciaremos Apache Tomcat:

    $ sudo systemctl restart tomcat9

    Y por último volveremos a ver si ahora nos funciona correctamente:

    $ sudo systemctl status tomcat9

  11. Apache Tomcat usa el puerto 8080 por defecto, para comprobar que todo está funcionando correctamente usaremos:

    $ ss -ltn

  12. Ahora debemos editar el fichero tomcat-users.xml donde debemos indicar nuestro usuario, contraseña y el o los roles que vamos a asignarle al usuario que pongamos. En este caso usaremos este como admin:

    $ sudo nano /etc/tomcat9/tomcat-users.xml
    $ <user username="usuario" password="usuario" roles="admin-gui,manager-gui"/>

  13. Actualizamos Apache Tomcat:

    $ sudo systemctl restart tomcat9

  14. Y por último comprobamos que todo funciones correctamente, para ello debemos poner en el navegador que deseemos lo siguiente:

    http://nuestra_ip:8080/manager/html

    En este caso usaremos Firefox, poniendo lo siguiente:

    $ firefox

    Introducimos el usuario y contraseña que pusimos en el documento XML y le damos a iniciar sesión

    Ya estamos dentro del gestor de aplicaciones que tengamos en nuestro Apache Tomcat
